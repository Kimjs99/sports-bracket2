# 학교스포츠클럽 축제 대진관리

학교스포츠클럽 축제의 종목별 대진 생성·경기 진행·결과 관리 웹앱입니다.  
토너먼트, 리그전, 조별리그+토너먼트 방식을 지원합니다.

**배포 주소:** https://kimjs99.github.io/sports-bracket2/

## 1. 처음 사용하는 학교 — 등록

1. 앱 접속 시 **학교/기관 선택** 화면이 표시됩니다.
2. 하단 **"새 학교/기관 등록"** 버튼을 누릅니다.
3. 다음 항목을 입력합니다.
   - **학교명**: 예) `서울중학교`
   - **식별자(영문)**: URL에 사용되는 고유 ID. 학교명을 입력하면 자동 생성됩니다. 영문·숫자·하이픈만 허용, 다른 학교와 중복 불가.
   - **비밀번호**: 이 학교의 관리자 비밀번호. **분실 시 복구 수단이 없으므로 반드시 기록해 두세요.**
4. **"등록하기"** 버튼을 누르면 계정이 생성되고 자동으로 로그인됩니다.

> 등록된 학교 데이터는 다른 학교와 완전히 격리됩니다. 다른 학교의 대진은 볼 수 없습니다.

---

## 2. 기존 학교 — 로그인

1. 앱 접속 시 **학교/기관 선택** 화면에서 목록을 스크롤하거나 검색창에 학교명을 입력합니다.
2. 학교를 선택하면 **비밀번호 입력 창**이 나타납니다.
3. 비밀번호를 입력하고 **"로그인"** 을 누릅니다.
4. 로그인 성공 시 홈 화면으로 이동합니다.

---

## 3. 대진 생성

1. 홈 화면 우측 상단 **"+ 새 대진 생성"** 버튼을 누릅니다.
2. **설정** 화면에서 아래 항목을 순서대로 선택합니다.

   | 항목 | 선택지 |
   |------|--------|
   | 학교급 | 초등 / 중등 / 고등 |
   | 성별 | 남성 / 여성 / 혼성 |
   | 종목 | 농구, 축구, 배구 등 22개 종목 |
   | 경기 방식 | 토너먼트 / 리그전 |

3. **팀 목록 입력** — 팀 이름을 직접 입력하거나, CSV·Excel 파일을 업로드합니다.
   - CSV/Excel 파일은 **한 열에 팀 이름** 형식이면 됩니다.
   - 템플릿 다운로드 버튼을 눌러 샘플 파일을 받을 수 있습니다.

4. **"대진 생성"** 버튼을 누릅니다.

### 경기 방식 상세

| 방식 | 조건 | 설명 |
|------|------|------|
| 토너먼트 | 팀 수 무관 | 단판 승부, 패자 탈락 |
| 리그전 (6팀 이하) | ≤6팀 | 모든 팀이 서로 한 번씩 대결, 무승부 허용 |
| 조별리그+토너먼트 | 리그전 선택 + 7팀 이상 | 자동 전환. 조별 리그 완료 후 상위 팀이 토너먼트 진출 |

---

## 4. 경기 일정 입력

1. 대진 화면에서 **"일정 입력"** 탭을 선택합니다.
2. 각 경기의 **날짜 / 시간 / 장소**를 입력합니다.
3. 입력한 일정은 대진표 카드에 표시됩니다.

---

## 5. 경기 결과 입력

1. 홈 화면에서 대진을 선택한 뒤 **"경기 진행"** 버튼을 누릅니다.
2. 라운드(또는 조)별 탭에서 경기를 확인합니다.
3. 각 경기의 **홈 팀 점수 / 어웨이 팀 점수**를 입력하고 **"결과 저장"** 을 누릅니다.

   - **토너먼트**: 동점 불가. 다음 라운드에 승자가 자동 배정됩니다.
   - **리그전**: 무승부 허용. 순위는 승점→득실차→득점 순으로 자동 계산됩니다.
   - **조별리그+토너먼트**: 조별 전 경기 완료 시 녹아웃 대진이 자동 생성됩니다.

4. 결과 수정이 필요하면 해당 경기 결과를 다시 입력하면 됩니다.  
   (조별 경기를 수정하면 녹아웃 대진이 초기화됩니다.)

---

## 6. 대진표 보기 및 다운로드

- 대진 화면에서 **"대진표"** 탭을 선택하면 브라켓 또는 순위표를 확인할 수 있습니다.
- **다운로드** 버튼(우상단)을 누르면 PNG / JPG / PDF 형식으로 저장할 수 있습니다.

---

## 7. 로그아웃

- 화면 **우하단 플로팅 바**의 학교명 버튼을 누릅니다.
- **"로그아웃"** 버튼을 누르면 로그아웃됩니다.
- 로그아웃 후 다른 학교로 로그인하면 이전 학교 데이터는 표시되지 않습니다.

---

## 8. 주의사항

- **비밀번호 분실 시 복구 불가**: 현재 이메일 기반 비밀번호 재설정이 구현되어 있지 않습니다. 비밀번호를 반드시 안전한 곳에 기록해 두세요.
- **데이터 격리**: 각 학교의 대진 데이터는 완전히 분리됩니다. 다른 학교 계정에서는 절대 조회되지 않습니다.
- **다크 모드**: 화면 우하단 플로팅 바의 해/달 아이콘으로 라이트/다크 모드를 전환할 수 있습니다.
- **모바일 지원**: 모바일·태블릿·데스크탑 모두 사용 가능합니다.

## 9. 관리자 참고사항

### 기술 스택

| 항목 | 내용 |
|------|------|
| 프론트엔드 | React 19 + Vite 8 + Tailwind CSS v4 |
| 인증/DB | Supabase (Auth + PostgreSQL + RLS) |
| 배포 | GitHub Pages (Kimjs99/sports-bracket2) |

### 로컬 개발 환경 설정

```bash
# 1. 의존성 설치
npm install

# 2. 환경변수 파일 생성 (.env)
VITE_SUPABASE_URL=https://tzdpfjypawavuclyxjeb.supabase.co
VITE_SUPABASE_ANON_KEY=<Supabase anon key>

# 3. 개발 서버 시작
npm run dev        # http://localhost:5173

# 4. 프로덕션 빌드
npm run build
```

### 신규 Supabase 프로젝트 설정

새 Supabase 프로젝트를 사용하는 경우 `SUPABASE_MIGRATION.sql`을 SQL 에디터에서 실행하세요.  
organizations 테이블과 tournaments 테이블의 RLS 정책이 생성됩니다.

### GitHub Actions 배포

`master` 브랜치에 push 하면 자동으로 배포됩니다 (약 50초 소요).  
GitHub Secrets에 다음 값이 등록되어 있어야 합니다.

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

### 데이터 구조

- **organizations 테이블**: 학교별 계정 정보 (slug, name, user_id)
- **tournaments 테이블**: 대진 데이터 (user_id 기반 RLS 격리)
- 학교별 로그인 계정은 `admin+{식별자}@school-bracket.app` 형식의 이메일을 사용합니다.
